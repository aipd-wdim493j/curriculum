Real-Time Node Web Social Sharing App

Alone or in groups, using the mad skillz you've acquired so far, craft a real-time web app that allows users to share something.

  • I am well aware that I just described nearly every social network in existence. Your job is to create a real-time social web experience with a niche market. Multi-player card game? Yes. Health information exchange? Oh yeah. In-browser, real-time Photoshop Tennis? Absolutely. A Designer's Porfolio? Nope.
  • Users should be able to register either using a username and password or via their existing Twitter or Facebook accounts.
  • The data stored in the app should survive through a restart of the Node app (Redis, CouchDB, MySQL, flat-files, whatever).
  • When data coming from multiple users is visualized, it should be visualized in real-time (with Socket.IO, Ajax, whatever). This might include graphs of user activity, win/loss leadboards, latest Photoshop Tennis composition, etc.
  • Using Express is suggested, but you can use any web app framework that suits you (or none at all).
  • The user experience should be well-thought-out, the UI should be well-designed and the JavaScript should be solid.
